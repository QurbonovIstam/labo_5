# Лаба 5 — Agile и Scrum

Проект представляет собой учебную модель простого приложения
для управления задачами. Цель проекта — продемонстрировать
применение Agile-подхода и Scrum-процесса с использованием GitHub Issues.
В рамках лабораторной работы был сформирован product backlog,
спланирован один спринт и смоделирован его ход.

---

## Product Backlog (Issues)

В рамках проекта были созданы следующие задачи:

1. Регистрация пользователя  
2. Авторизация пользователя  
3. Просмотр списка задач  
4. Создание новой задачи  
5. Редактирование задачи  
6. Удаление задачи  
7. Хранение данных задач  
8. Документация проекта

Каждый issue содержит:
- краткое описание задачи
- критерии готовности (Definition of Done)
- приоритет (high / medium / low)
- оценку сложности (Story points)
- принадлежность к спринту (label `sprint-1`)

---

## Sprint 1

В спринт были включены следующие задачи:

| Задача | Label | Статус | Приоритет | Story points |
|--------|-------|--------|-----------|--------------|
| Регистрация пользователя | sprint-1 | Closed | high | 3 |
| Авторизация пользователя | sprint-1 | Closed | high | 2 |
| Просмотр списка задач | sprint-1 | Open / todo | medium | 2 |
| Создание новой задачи | sprint-1 | Open / in-progress | high | 3 |
| Редактирование задачи | sprint-1 | Open / todo | low | 5 |

Остальные задачи остались в беклоге для следующих спринтов:
- Удаление задачи  
- Хранение данных задач  
- Документация проекта  

---

## Sprint Review

В рамках спринта были завершены задачи:
- Регистрация пользователя
- Авторизация пользователя

Также была начата работа над задачей:
- Создание новой задачи

Задачи, которые не были завершены:
- Просмотр списка задач
- Редактирование задачи

По итогу спринта была доставлена базовая функциональность
системы: регистрация, авторизация и возможность создавать задачи.

---

## Sprint Retrospective

**Что прошло хорошо:**  
- Чётко сформулирован product backlog  
- Быстрое планирование задач спринта  
- Правильная расстановка приоритетов  

**Что пошло не так:**  
- Сложно было оценить объём задач  
- Некоторые задачи остались незавершёнными  

**Что можно улучшить в следующем спринте:**  
- Более точная оценка сложности задач (Story points)  
- Уменьшение количества задач в спринте  
- Следить за тем, чтобы все задачи были распределены по статусам
